### Task 1: Fix OAuth Token Cache Staleness (Bug)

**Problem:** `AnthropicLLM._resolvedKey` is set once and never refreshed. After token expires (or refreshes overnight), the cached Anthropic SDK client still uses the old token → 401 → "authorization expired" error. But the token IS valid (it auto-refreshed) — the cache is just stale.

**Files:**
- Modify: `server/utils/AiProviders/anthropic/index.js`

**Fix:** Remove the aggressive caching. Instead of caching `_resolvedKey` and `this.anthropic` forever, check token freshness on every request. The `getValidAccessToken()` call is cheap (reads a file, checks expiry) — no need to cache.

```javascript
// In #resolveAuth() — remove the cache check at the top:
// DELETE: if (this._resolvedKey) return this._resolvedKey;
// REPLACE with: check if cached token is still the same as current

async #resolveAuth() {
  // Always check for fresh OAuth token (file read, cheap)
  const oauthToken = await getValidAccessToken();
  if (oauthToken) {
    // If token changed (refresh happened), invalidate client cache
    if (this._resolvedKey?.key !== oauthToken) {
      this.anthropic = null; // Force new client on next call
    }
    this._resolvedKey = { key: oauthToken, isOAuth: true };
    this.isOAuth = true;
    return this._resolvedKey;
  }

  // Fall back to manual API key
  const manualKey = process.env.ANTHROPIC_API_KEY;
  if (!manualKey || manualKey === "sk-ant-oauth-managed") {
    throw new Error("No Anthropic authentication found.");
  }
  this._resolvedKey = { key: manualKey, isOAuth: false };
  return this._resolvedKey;
}
```

Also fix `#getClient()` to not cache when OAuth (token can change):
```javascript
async #getClient() {
  const { key, isOAuth } = await this.#resolveAuth();
  // Rebuild client if not cached or if OAuth (token may have refreshed)
  if (!this.anthropic) {
    // ... create client as before
  }
  return this.anthropic;
}
```

Same fix needed in `server/utils/agents/aibitat/providers/anthropic.js` — the constructor loads the token once. For the agent, the instance is short-lived (per-request), so this is less critical, but the `loadTokens()` call should use `getValidAccessToken()` for auto-refresh.

**Commit:** `fix: OAuth token cache invalidation — refresh on every request, no stale clients`

### Task 2: Fix Agent tool_use/tool_result Message Ordering (Bug)

**Problem:** After 4+ consecutive MCP tool calls, the Anthropic API returns 400: `"tool_use" ids were found without "tool_result" blocks`. This happens because `AnthropicProvider.#prepareMessages()` doesn't always pair tool_use with tool_result correctly when the agent framework builds long conversation histories.

**Files:**
- Modify: `server/utils/agents/aibitat/providers/anthropic.js` — `#prepareMessages()` method

**Root Cause:** The `#prepareMessages()` method normalizes `function` role messages to `tool_result` format, but it only looks at the PREVIOUS message for the `tool_use` block. In multi-step agent loops, tool_use blocks can be nested in content arrays and the simple `prevMessage` check misses them. Also, consecutive same-role merging can break the strict alternation.

**Fix:** Add a validation pass at the end of `#prepareMessages()` that ensures every `tool_use` in an assistant message has a matching `tool_result` in the next user message:

```javascript
// After building processedMessages, add validation:
// Ensure tool_use/tool_result pairing
for (let i = 0; i < processedMessages.length - 1; i++) {
  const msg = processedMessages[i];
  if (msg.role !== "assistant" || !Array.isArray(msg.content)) continue;
  
  const toolUses = msg.content.filter(b => b.type === "tool_use");
  if (toolUses.length === 0) continue;
  
  const nextMsg = processedMessages[i + 1];
  if (!nextMsg || nextMsg.role !== "user") {
    // Insert a synthetic tool_result for each orphaned tool_use
    const toolResults = toolUses.map(tu => ({
      type: "tool_result",
      tool_use_id: tu.id,
      content: "Tool execution completed.",
    }));
    processedMessages.splice(i + 1, 0, {
      role: "user",
      content: toolResults,
    });
  }
}
```

**Commit:** `fix: agent tool_use/tool_result message pairing — prevent orphaned blocks`

### Task 4: Security — Add Auth to OAuth Endpoints [CRITICAL]

**Problem:** All `/api/anthropic-oauth/*` endpoints have zero authentication. Anyone on the network can logout a user's session or start rogue OAuth flows.

**Files:**
- Modify: `server/endpoints/anthropicOAuth.js`

**Fix:** Add `validatedRequest` middleware to all OAuth endpoints:
```javascript
const { validatedRequest } = require("../utils/middleware/validatedRequest");

// Before each route:
app.get("/anthropic-oauth/start", [validatedRequest], async (_req, res) => { ... });
app.get("/anthropic-oauth/status", [validatedRequest], async (_req, res) => { ... });
app.post("/anthropic-oauth/refresh", [validatedRequest], async (_req, res) => { ... });
app.post("/anthropic-oauth/logout", [validatedRequest], async (_req, res) => { ... });
app.get("/anthropic-oauth/token", [validatedRequest], async (_req, res) => { ... });
```

**Commit:** `security: add authentication to all OAuth endpoints`

### Task 5: Security — Fix PKCE State Parameter [CRITICAL]

**Problem:** The OAuth `state` parameter reuses the PKCE `verifier`, collapsing CSRF protection. State should be a separate random value.

**Files:**
- Modify: `server/utils/AiProviders/anthropic/oauth.js`

**Fix:** Generate a separate state parameter:
```javascript
async function startOAuthFlow() {
  const { verifier, challenge } = generatePKCE();
  const state = crypto.randomBytes(32).toString("base64url"); // Separate from verifier
  
  const callbackServer = await startCallbackServer(state); // Validate state on callback
  
  const authParams = new URLSearchParams({
    // ...
    code_challenge: challenge,
    state: state, // NOT verifier
  });
  
  activeSession = { verifier, state, callbackServer, ... };
}
```

And in `startCallbackServer`, validate `state` against the stored value, then use the separately stored `verifier` for the token exchange.

**Commit:** `security: separate PKCE verifier from OAuth state — restore CSRF protection`

### Task 6: Security — Encrypt OAuth Tokens at Rest [HIGH]

**Problem:** `anthropic_oauth.json` stores access_token and refresh_token as plaintext JSON.

**Files:**
- Modify: `server/utils/AiProviders/anthropic/tokenStorage.js`

**Fix:** Use AnythingLLM's existing `EncryptionManager` to encrypt tokens before writing and decrypt on read:
```javascript
const EncryptionManager = require("../../EncryptionManager");
const encryptionManager = new EncryptionManager();

function saveTokens(tokens) {
  const encrypted = encryptionManager.encrypt(JSON.stringify(tokens));
  fs.writeFileSync(tokenPath, encrypted, "utf-8");
}

function loadTokens() {
  const raw = fs.readFileSync(tokenPath, "utf-8");
  const decrypted = encryptionManager.decrypt(raw);
  return JSON.parse(decrypted);
}
```

**Commit:** `security: encrypt OAuth tokens at rest via EncryptionManager`
